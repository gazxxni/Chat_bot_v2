# 컴퓨터공학과 교과과정 안내 챗봇

컴퓨터공학과 신입생 및 재학생의 성공적인 수강 계획을 돕기 위해 설계된 RAG(검색 증강 생성) 기반 챗봇 프로젝트입니다.

## 1. 프로젝트 개요

* **프로젝트 유형:** [성결대학교 컴퓨터공학과] 학부생 연구 프로젝트
* **수행 기간:** 2025년 1월 ~ 2025년 6월
* **팀원 (Contributors):** 천민우, 정세준, 김시현, 신혜민

## 2. 프로젝트 배경 및 목표

대학교 신입생, 특히 전공자율선택제를 통해 입학한 학생들은 복잡한 학과 교육과정 문서를 마주했을 때 수강 계획 수립에 큰 어려움을 겪습니다.

* 과목 간의 복잡한 **선후수 관계**
* "인공지능", "데이터베이스" 등 **진로 트랙별** 이수 과정
* 과목이 **진로와 어떻게 연계**되는지에 대한 안내 부족

이러한 정보 탐색의 어려움은 학생들의 학습 동기를 저하하고 비효율적인 수강 계획을 초래할 수 있습니다.

본 프로젝트는 이러한 문제를 해결하기 위해, 학과 교육과정 PDF 문서를 지식 베이스로 활용하는 챗봇 시스템을 구축하는 것을 목표로 합니다. 사용자는 정적인 웹페이지를 탐색하는 대신, 자연어 질문을 통해 자신에게 필요한 맞춤형 답변을 실시간으로 제공받을 수 있습니다.

## 3. 주요 기능

* **자연어 질의응답:** "인공지능 관련 과목 추천해줘" 또는 "자료구조 선수과목이 뭐야?"와 같은 자연어 질문을 이해하고 답변합니다.
* **RAG 기반 정보 제공:** 챗봇이 임의로 답변을 생성하는 것이 아니라(환각), `역량중심 교육과정 보고서.pdf` 원문에서 질문과 가장 관련성이 높은 내용을 검색하여(Top-K=10), 이를 근거로 답변을 생성합니다.
* **맞춤형 페르소나:** 신입생에게 친근하고 명확하게 설명해 주는 "조교" 또는 "상담 선생님"의 페르소나를 적용하여 답변의 가독성과 이해도를 높였습니다.

## 4. 시스템 아키텍처 (RAG)

본 시스템은 검색 증강 생성(RAG) 아키텍처를 기반으로 합니다.

1.  **데이터 준비 (Ingest):**
    * 원본 PDF 문서(`역량중심...pdf`)를 `semantic_ingest_combined.py` 스크립트를 통해 의미 단위(Semantic Chunks)로 분할합니다.
    * OpenAI의 `text-embedding-3-small` 모델을 사용하여 각 텍스트 조각을 벡터로 변환(임베딩)합니다.
    * 변환된 벡터를 `ChromaDB`에 저장하여 지식 베이스를 구축합니다.

2.  **응답 생성 (Runtime):**
    * **[Frontend (React)]** 사용자가 `http://localhost:3000`에서 질문을 입력합니다.
    * **[Backend (FastAPI)]** `/ask` 엔드포인트가 질문을 받아, OpenAI 임베딩 모델을 사용해 사용자 질문을 벡터로 변환합니다.
    * **[Vector DB (ChromaDB)]** 질문 벡터와 가장 유사한(가장 관련성 높은) 텍스트 조각 10개(Top-K=10)를 DB에서 검색합니다.
    * **[AI Model (OpenAI)]** 검색된 10개의 텍스트(Context)와 원본 질문을 프롬프트에 조합하여 `gpt-4o` 모델에게 전달합니다.
    * **[Response]** AI 모델이 생성한 최종 답변을 다시 프론트엔드로 전송하여 사용자 화면에 표시합니다.

## 5. 기술 스택

* **Frontend:** React
* **Backend:** FastAPI (Python)
* **Vector Database:** ChromaDB
* **Embedding Model:** OpenAI `text-embedding-3-small`
* **Generation Model:** OpenAI `gpt-4o`
